## Why

RHESS has unit and integration tests but no end-to-end validation of the full deployment lifecycle — build → container → deploy to OpenShift → live traffic. Onboarding to OpenShift CI gives every PR a real cluster-based gate that exercises the container image, Kubernetes manifests, and HTTP API in a production-like environment before merge.

## What Changes

- **New `.ci/` directory in rhess**: Shell script (`openshift_e2e.sh`) that deploys RHESS onto the provisioned cluster, runs API smoke tests, verifies the `.well-known/` discovery endpoint, and checks readiness probes
- **New ci-operator config in `openshift/release`**: `ci-operator/config/redhat-ai-dev/rhess/redhat-ai-dev-rhess-main.yaml` using the `aws-devfile` cluster profile (identical pattern to `devfile/registry`)
- **OWNERS file in the release repo**: `ci-operator/config/redhat-ai-dev/rhess/OWNERS` listing maintainers
- **Prow job configs** (generated): `ci-operator/jobs/redhat-ai-dev/rhess/` pre/post-submit jobs generated by `make update` in the release repo

## Capabilities

### New Capabilities

- `e2e-openshift`: Shell-based end-to-end test suite that deploys RHESS on a live OpenShift cluster and validates the full operational flow

### Modified Capabilities

None

## Canonical Touchpoints

Affected long-lived specs under `openspec/specs/`:
- `container-deployment` — the e2e tests exercise the deployment manifests; no requirement changes, informational dependency only

No files under `specifications/` are affected.

Change type: **feature-spec**

## Impact

- **New files in rhess repo**: `.ci/openshift_e2e.sh`, minor additions to `.ci/` if a separate Dockerfile variant is needed (none expected — root `Dockerfile` is already suitable)
- **New files in `openshift/release`**: CI operator config YAML, OWNERS, generated Prow job YAML
- **No API changes**: E2e tests are consumers of existing endpoints, not modifiers
- **Dependencies**: `aws-devfile` cluster profile already defined in OpenShift CI; no new profile required
- **CI gates**: New `e2e-openshift` test job runs on every PR to `redhat-ai-dev/rhess` main branch
